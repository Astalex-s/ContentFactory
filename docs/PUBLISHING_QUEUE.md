# Очередь публикаций — Планирование и управление

## Обзор

Система планирования публикаций позволяет:
- Выбирать несколько единиц контента (видео/изображения)
- Планировать публикации в разные социальные сети
- Указывать время публикации для каждого контента
- Управлять очередью публикаций (просмотр, фильтрация, отмена)

---

## Backend API

### 1. Получение списка публикаций

**Endpoint:** `GET /publish/`

**Query параметры:**
- `status` (optional): `pending`, `processing`, `published`, `failed`
- `platform` (optional): `youtube`, `vk`, `tiktok`
- `limit` (optional): 1-100, default 50
- `offset` (optional): ≥0, default 0

**Response:**
```json
{
  "total": 42,
  "items": [
    {
      "id": "uuid",
      "content_id": "uuid",
      "platform": "youtube",
      "account_id": "uuid",
      "scheduled_at": "2026-03-01T15:00:00Z",
      "status": "pending",
      "error_message": null,
      "platform_video_id": null,
      "created_at": "2026-02-27T10:00:00Z"
    }
  ],
  "limit": 50,
  "offset": 0
}
```

**Статусы:**
- `pending` — ожидает публикации
- `processing` — в процессе загрузки
- `published` — успешно опубликовано
- `failed` — ошибка при публикации

---

### 2. Планирование одной публикации

**Endpoint:** `POST /publish/{content_id}`

**Rate limit:** 5 запросов в минуту

**Request body:**
```json
{
  "platform": "youtube",
  "account_id": "uuid",
  "scheduled_at": "2026-03-01T15:00:00Z",
  "title": "Название видео",
  "description": "Описание видео"
}
```

**Поля:**
- `platform` (required): `youtube`, `vk`, `tiktok`
- `account_id` (required): UUID подключённого аккаунта
- `scheduled_at` (optional): дата/время публикации (UTC). Если не указано — публикация сразу
- `title` (optional): заголовок видео (max 100 символов)
- `description` (optional): описание видео (max 5000 символов)

**Validation:**
- `scheduled_at` должно быть в будущем (если указано)
- Контент должен существовать
- Аккаунт должен быть подключён
- Платформа аккаунта должна совпадать с указанной

**Response:** объект публикации (см. выше)

---

### 3. Массовое планирование публикаций

**Endpoint:** `POST /publish/bulk`

**Rate limit:** 3 запроса в минуту

**Request body:**
```json
{
  "publications": [
    {
      "content_id": "uuid",
      "platform": "youtube",
      "account_id": "uuid",
      "scheduled_at": "2026-03-01T15:00:00Z",
      "title": "Название 1",
      "description": "Описание 1"
    },
    {
      "content_id": "uuid",
      "platform": "vk",
      "account_id": "uuid",
      "scheduled_at": "2026-03-01T16:00:00Z",
      "title": "Название 2",
      "description": "Описание 2"
    }
  ]
}
```

**Ограничения:**
- Минимум 1, максимум 50 публикаций за запрос
- Все поля валидируются как в одиночной публикации
- Если хотя бы одна публикация невалидна — весь запрос отклоняется

**Response:**
```json
{
  "created_count": 2,
  "publications": [
    { /* объект публикации 1 */ },
    { /* объект публикации 2 */ }
  ]
}
```

---

### 4. Отмена публикации

**Endpoint:** `DELETE /publish/{id}`

**Описание:** Отменяет/удаляет публикацию из очереди

**Ограничения:**
- Можно отменить только публикации со статусом `pending`
- Публикации в процессе (`processing`) или завершённые (`published`, `failed`) отменить нельзя

**Response:**
```json
{
  "message": "Публикация отменена"
}
```

**Ошибки:**
- `404` — публикация не найдена или уже обработана

---

### 5. Проверка статуса публикации

**Endpoint:** `GET /publish/status/{id}`

**Response:**
```json
{
  "id": "uuid",
  "status": "published",
  "error_message": null,
  "platform_video_id": "dQw4w9WgXcQ"
}
```

## Frontend — Страница публикаций

### Путь
`/publishing` → `PublishingPage.tsx`

### Функционал

#### 1. Список публикаций
- Таблица со всеми запланированными публикациями
- Колонки:
  - Content ID (сокращённый)
  - Платформа (бейдж)
  - Запланировано (дата + время, индикатор просроченных)
  - Статус (бейдж с цветом)
  - Создано
  - Действия (кнопки)

#### 2. Фильтры
- **Статус:** все / ожидает / в процессе / опубликовано / ошибка
- **Платформа:** все / YouTube / VK / TikTok
- Кнопка "Сбросить фильтры"

#### 3. Действия над публикациями
- **Отменить** — для публикаций со статусом `pending`
- **Открыть** — для опубликованных (ссылка на видео на платформе)

#### 4. Создание новых публикаций
Кнопка "+ Запланировать публикации" открывает двухшаговый процесс:

**Шаг 1: Выбор контента** (`ContentSelector.tsx`)
- Модальное окно со списком доступного контента
- Фильтры:
  - Тип контента: все / видео / изображения
  - Платформа: все / YouTube / VK / TikTok
- Показываются только:
  - Контент типа `video` или `image`
  - Со статусом `ready`
- Множественный выбор (чекбоксы)
- Кнопка "Выбрать все" / "Снять выделение"
- Счётчик выбранного контента

**Шаг 2: Настройка публикаций** (`SchedulePublicationModal.tsx`)
- Для каждого выбранного контента:
  - **Платформа** (dropdown): YouTube / VK / TikTok
  - **Аккаунт** (dropdown): список подключённых аккаунтов для выбранной платформы
  - **Дата и время** (datetime-local): по умолчанию +1 час для каждого следующего
  - **Заголовок** (text, optional, max 100): можно оставить пустым для автоматического
  - **Описание** (textarea, optional, max 5000): по умолчанию — текст контента
- Валидация перед отправкой:
  - Все публикации должны иметь выбранную платформу и аккаунт
  - Время должно быть в будущем
- Кнопка "Запланировать" отправляет массовый запрос

## Компоненты

### ContentSelector.tsx
**Props:**
- `isOpen: boolean` — показывать модалку
- `onClose: () => void` — закрыть модалку
- `onSelect: (selected: GeneratedContent[]) => void` — callback с выбранным контентом

**Состояние:**

**Методы:**
- `loadContent()` — загрузка контента через API
- `toggleSelect(id)` — переключение выбора элемента

- `handleConfirm()` — подтверждение выбора

## Workflow пользователя

### Планирование публикаций

1. Пользователь открывает страницу `/publishing`
2. Нажимает "+ Запланировать публикации"
3. Открывается `ContentSelector`:
   - Видит список всего готового контента (видео/изображения)
   - Применяет фильтры по типу и платформе
   - Выбирает нужный контент (чекбоксы)
   - Нажимает "Продолжить (N)"
4. Открывается `SchedulePublicationModal`:
   - Для каждого выбранного контента настраивает:
     - Платформу (YouTube/VK/TikTok)
     - Аккаунт (из подключённых)
     - Время публикации
     - Заголовок и описание (опционально)
   - Нажимает "Запланировать"
5. Система:
   - Валидирует все данные
   - Создаёт записи в очереди публикаций
   - Для публикаций "сейчас" — запускает фоновую задачу
   - Для будущих — ставит в очередь
6. Пользователь видит обновлённую таблицу публикаций

### Управление публикациями

1. Просмотр списка:
   - Все публикации в таблице
   - Фильтрация по статусу и платформе
   - Сортировка по дате планирования
2. Отмена публикации:
   - Кнопка "Отменить" для `pending`
   - Подтверждение действия
   - Удаление из очереди
3. Просмотр результата:
   - Для опубликованных — кнопка "Открыть"
   - Переход на видео на платформе
   - Для ошибок — отображение сообщения об ошибке

## Обработка публикаций

### Фоновые задачи

Публикации обрабатываются через `BackgroundTasks` (MVP) или Celery (production):

1. **Немедленные публикации** (`scheduled_at` ≤ now):
   - Запускаются сразу после создания записи
   - Статус: `pending` → `processing` → `published`/`failed`

2. **Отложенные публикации** (`scheduled_at` > now):
   - Остаются в статусе `pending`
   - Обрабатываются cron: `POST /publish/process-pending` (каждую минуту)
   - Эндпоинт выбирает `pending` записи с `scheduled_at` ≤ now и запускает обработку

### Настройка cron на сервере

Для отложенных публикаций добавьте в crontab (`crontab -e`):

```
* * * * * curl -sS -X POST http://127.0.0.1:8000/publish/process-pending
```

Cron проверяет БД каждую минуту и публикует видео, у которых наступило время.

### Процесс публикации

1. Проверка существования файла
2. Обновление токена (если истёк)
3. Для длинных видео (≥60 сек):
   - Добавление ссылки на товар в описание
4. Для всех видео:
   - QR-код с ссылкой уже в видео (добавлен при генерации)
5. Загрузка через провайдер платформы
6. Обновление статуса в БД

## Безопасность и ограничения

### Rate Limiting
- Одиночная публикация: 5/минуту
- Массовая публикация: 3/минуту
- Защита от перегрузки API платформ

### Валидация
- Время публикации в будущем
- Существование контента и аккаунта
- Соответствие платформы аккаунта
- Лимиты на длину заголовка и описания

### Ограничения
- Максимум 50 публикаций за один массовый запрос
- Отменить можно только `pending` публикации
- Токены автоматически обновляются перед публикацией
