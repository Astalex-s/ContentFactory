# ContentSelector — Функционал выбора контента

## Обзор

ContentSelector — модальное окно для выбора контента перед планированием публикаций. Поддерживает все типы контента с визуальными превью и удобной навигацией.

---

## Типы контента

### 1. Видео 🎥
- **Превью:** Первый кадр видео с кнопкой воспроизведения
- **Размер превью:** 120x90px
- **Отображение:** Миниатюра видео + иконка play
- **Текст:** Описание видео (первые 150 символов)

### 2. Изображения 🖼️
- **Превью:** Само изображение
- **Размер превью:** 120x90px
- **Отображение:** Миниатюра изображения
- **Текст:** Описание изображения (первые 150 символов)

### 3. Текст 📝
- **Превью:** Иконка документа (📝)
- **Размер превью:** 120x90px
- **Отображение:** Иконка на сером фоне
- **Текст:** Полный текст с возможностью развернуть

## Функционал

### Фильтрация

**Тип контента:**
- Все
- Видео
- Изображения
- Текст

**Платформа:**
- Все
- YouTube
- VK
- TikTok

### Выбор контента

**Множественный выбор:**
- Чекбокс справа от каждого элемента
- Клик по элементу переключает выбор
- Кнопка "Выбрать все" / "Снять выделение"

**Счётчик:**
- Внизу отображается: "Выбрано: N из M"
- Кнопка "Продолжить" показывает количество выбранных

### Работа с текстом

**Длинный текст (>150 символов):**
- По умолчанию показываются первые 150 символов + "..."
- Кнопка "Развернуть" показывает полный текст
- Кнопка "Свернуть" возвращает к краткому виду
- Текст сохраняет форматирование (переносы строк)

**Короткий текст (≤150 символов):**
- Показывается полностью
- Кнопка развернуть не отображается

### Состояния

**Не выбрано:**
- Граница: серая (#E5E7EB)
- Фон: белый
- Чекбокс: пустой

**Выбрано:**
- Граница: синяя (#4F46E5)
- Фон: светло-синий (rgba(79, 70, 229, 0.1))
- Чекбокс: синий с галочкой

**Hover:**
- Плавный переход (0.2s)

## Превью медиа

### Видео

```tsx
<video
  src={`${API_BASE_URL}/content/media/${item.file_path}`}
  style={{
    width: "100%",
    height: "100%",
    objectFit: "cover",
  }}
/>
```

**Особенности:**
- Показывается первый кадр (poster)
- Иконка play по центру
- Не воспроизводится автоматически
- Клик по превью не воспроизводит видео (только выбор)

### Изображения

```tsx
<img
  src={`${API_BASE_URL}/content/media/${item.file_path}`}
  alt="Preview"
  style={{
    width: "100%",
    height: "100%",
    objectFit: "cover",
  }}
/>
```

**Особенности:**
- Масштабируется с сохранением пропорций
- Обрезается по центру (object-fit: cover)

### Текст

```tsx
<div style={{
  fontSize: "36px",
  color: colors.gray[500],
}}>
  📝
</div>
```

**Особенности:**
- Иконка документа
- Серый цвет
- Центрирование

## Использование

### Открытие

```tsx
<ContentSelector
  isOpen={showContentSelector}
  onClose={() => setShowContentSelector(false)}
  onSelect={handleContentSelected}
/>
```

### Props

- `isOpen: boolean` — показывать модалку
- `onClose: () => void` — закрыть модалку
- `onSelect: (selected: GeneratedContent[]) => void` — callback с выбранным контентом

### Callback

```tsx
const handleContentSelected = (content: GeneratedContent[]) => {
  console.log("Выбрано контента:", content.length);
  // Переход к следующему шагу (настройка публикаций)
  setSelectedContent(content);
  setShowScheduleModal(true);
};
```

## API

### Загрузка контента

**Endpoint:** `GET /content/all?page=1&page_size=100`

**Фильтрация на клиенте:**
- **Видео и изображения:** только статус `ready` (медиафайлы должны быть сгенерированы)
- **Текст:** статус `ready` или `draft` (текст всегда готов к публикации)
- Все типы: video, image, text

**Логика фильтрации:**
```typescript
const filtered = data.items.filter((item) => {
  if (item.content_type === "text") {
    return item.status === "ready" || item.status === "draft";
  }
  return item.status === "ready";
});
```

**Почему текст может быть draft:**
- Текстовый контент создаётся сразу в финальном виде
- Не требует дополнительной обработки (в отличие от видео/изображений)
- Может быть отредактирован пользователем (статус draft)
- Всегда готов к публикации

## Примеры использования

### Пример 1: Выбор видео для YouTube

1. Открыть ContentSelector
2. Фильтр "Тип контента" → Видео
3. Фильтр "Платформа" → YouTube
4. Выбрать 3 видео
5. Нажать "Продолжить (3)"

### Пример 2: Выбор текста для постов

1. Открыть ContentSelector
2. Фильтр "Тип контента" → Текст
3. Развернуть текст для просмотра
4. Выбрать нужные тексты
5. Нажать "Продолжить (N)"

### Пример 3: Выбор всего контента

1. Открыть ContentSelector
2. Нажать "Выбрать все"
3. Нажать "Продолжить (N)"

## Технические детали

### Состояние компонента

```tsx
const [content, setContent] = useState<GeneratedContent[]>([]);
const [selected, setSelected] = useState<Set<string>>(new Set());
const [expandedText, setExpandedText] = useState<Set<string>>(new Set());
const [filter, setFilter] = useState({ type: "all", platform: "all" });
```

### Методы

- `loadContent()` — загрузка контента через API
- `toggleSelect(id)` — переключение выбора элемента
- `toggleSelectAll()` — выбрать/снять все
- `toggleExpandText(id)` — развернуть/свернуть текст
- `handleConfirm()` — подтверждение выбора

### Оптимизация

- Загрузка контента только при открытии модалки
- Сброс выбора при закрытии
- Фильтрация на клиенте (без лишних запросов)
- Lazy loading для превью (браузер загружает по мере прокрутки)
