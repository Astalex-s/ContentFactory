# Changelog — Система планирования публикаций

## Дата: 27 февраля 2026

### Добавлен функционал массового планирования публикаций

Реализована полноценная система управления очередью публикаций с возможностью выбора контента, настройки времени и массового планирования.

---

## Backend изменения

### 1. Расширен репозиторий PublicationQueueRepository

**Файл:** `backend/app/repositories/publication_queue.py`

**Новые методы:**
- `get_all(status, platform, limit, offset)` — получение списка публикаций с фильтрами
- `count_all(status, platform)` — подсчёт публикаций с фильтрами
- `delete(queue_id)` — удаление публикации
- `bulk_create(publications)` — массовое создание публикаций

### 2. Расширен сервис PublicationService

**Файл:** `backend/app/services/publication_service.py`

**Новые методы:**
- `get_publications(status, platform, limit, offset)` — получение списка с фильтрами
- `count_publications(status, platform)` — подсчёт публикаций
- `bulk_schedule_publications(publications, background_tasks)` — массовое планирование
- `cancel_publication(queue_id)` — отмена публикации (только pending)

**Логика:**
- Валидация всех контентов и аккаунтов перед созданием
- Проверка соответствия платформы аккаунта
- Автоматический запуск немедленных публикаций через BackgroundTasks

### 3. Новые Pydantic схемы

**Файл:** `backend/app/schemas/publish.py`

**Добавлены схемы:**
- `PublicationItem` — элемент для массового планирования
- `BulkPublishRequest` — запрос массового планирования (1-50 публикаций)
- `BulkPublishResponse` — ответ с созданными публикациями
- `PublicationListResponse` — список публикаций с пагинацией

**Валидация:**
- `scheduled_at` должно быть в будущем
- Ограничение 1-50 публикаций за запрос
- Паттерны для platform (youtube|vk|tiktok)

### 4. Новые API endpoints

**Файл:** `backend/app/routers/publish.py`

**Добавлены endpoints:**

#### GET /publish/
Получение списка публикаций с фильтрами
- Query параметры: status, platform, limit (1-100), offset
- Response: PublicationListResponse с total, items, limit, offset

#### POST /publish/bulk
Массовое планирование публикаций
- Rate limit: 3 запроса в минуту
- Body: BulkPublishRequest (1-50 публикаций)
- Response: BulkPublishResponse с created_count и списком публикаций
- Валидация всех публикаций перед созданием

#### DELETE /publish/{id}
Отмена/удаление публикации
- Можно отменить только публикации со статусом pending
- Response: {message: "Публикация отменена"}
- 404 если публикация не найдена или уже обработана

**Props:**

- `onClose: () => void` — закрыть модалку

## Технические детали

### Rate Limiting
- Одиночная публикация: 5/минуту (без изменений)
- Массовая публикация: 3/минуту (новое)

### Ограничения
- Максимум 50 публикаций за один массовый запрос
- Заголовок: max 100 символов
- Описание: max 5000 символов
- Отменить можно только публикации со статусом pending

### Валидация
- Время публикации должно быть в будущем
- Контент должен существовать
- Аккаунт должен быть подключён
- Платформа аккаунта должна совпадать с выбранной

### Фоновые задачи
- Немедленные публикации (scheduled_at ≤ now) запускаются сразу
- Отложенные публикации ждут своего времени
- Статусы: pending → processing → published/failed

## Workflow пользователя

### Планирование публикаций

1. Открыть страницу `/publishing`
2. Нажать "+ Запланировать публикации"
3. **ContentSelector:**
   - Выбрать контент (фильтры по типу и платформе)
   - Отметить нужные элементы
   - Нажать "Продолжить (N)"
4. **SchedulePublicationModal:**
   - Для каждого контента настроить:
     - Платформу и аккаунт
     - Дату и время
     - Заголовок и описание (опционально)
   - Нажать "Запланировать"
5. Система создаёт публикации и обновляет таблицу

### Управление публикациями

1. Просмотр списка с фильтрами
2. Отмена публикаций (pending)
3. Просмотр результатов (ссылки на опубликованные видео)
4. Мониторинг статусов

## Тестирование

### Ручное тестирование

1. **Выбор контента:**
   - Проверить фильтры
   - Проверить множественный выбор
   - Проверить "Выбрать все"

2. **Настройка публикаций:**
   - Проверить выбор платформы и аккаунта
   - Проверить валидацию времени
   - Проверить лимиты на заголовок и описание

3. **Массовое планирование:**
   - Создать 5-10 публикаций
   - Проверить создание в БД
   - Проверить немедленные публикации

4. **Управление:**
   - Проверить фильтры
   - Отменить публикацию
   - Открыть опубликованное видео

### API тестирование

```bash
# Получение списка
curl http://localhost:8000/publish/?status=pending&limit=10

# Массовое планирование
curl -X POST http://localhost:8000/publish/bulk \
  -H "Content-Type: application/json" \
  -d '{"publications": [...]}'

# Отмена публикации
curl -X DELETE http://localhost:8000/publish/{id}
```

## Известные ограничения

1. **TikTok:** OAuth работает, но загрузка видео недоступна (нет публичного API)
2. **Редактирование:** нельзя редактировать запланированные публикации (только отмена и создание новой)
3. **Повторяющиеся публикации:** нет поддержки расписаний (ежедневно, еженедельно)
4. **Уведомления:** нет email/push уведомлений о статусе публикаций
